Blocks, Procs, and Lambdas

November 18, 2014

In this blog post I will discuss Blocks, Procs, and Lambdas, the three similar but distinct ways that Ruby allows you to pass commands to a method akin to the way you might pass arguments to a method. These features are known as closures in Computer Science.

Blocks

Blocks are the simplest closure in ruby and are good to use if you do not plan on reusing the code as they cannot be saved. I also recommend that you use the lowercase when writing "block" as block is just an instance of the Proc class which I will get to in a moment and is not a class in its own right.

Procs

Procs which is short for procedure. Differ from blocks in that they can are can be used more than once. Unlike blocks you can also pass multiple Procs to one method which can be handy. I recommend using uppercase when writing Proc.new because it is a real class in Ruby.

Lambdas

Unlike blocks and Procs, Lambdas act more or less like anonymous methods and therefore check the number of arguments passed in and do not return automatically but instead keep running.
