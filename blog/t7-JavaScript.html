JavaScript Objects compared with Ruby Hashes

November 24, 2014

Today I'm going to write a little about the differences I see between Objects in JavaScript and Hashes in Ruby. As I've discussed in a previous post Ruby Hashes are a data structure which is made up of one or many key and value pairs. The key is the first piece of information and the second is the value corresponding to that key. These are great when you want to store information with labels. See the code below for an example of a hash in Ruby with the name daniel_info. Note in the example that hashes can have arrays and other hashes nested within them. Also notice the symbol notation used for some of the keys in the hash.

            daniel_info = {
            home_town: "Palo Alto",
            full_name: "Daniel Kelley",
            "age" => 25,
            fav_food: "ice cream",
            places_lived: ["Palo Alto", "Providence", "Phnom Penh"]
            }

            puts daniel_info[:full_name]      # puts "Daniel Kelley"
            puts daniel_info["age"]           # puts 25
            puts daniel_info.fav_food         # this does not work because Ruby thinks fav_food is a method.

While you might be thinking that JavaScript Objects are likely more akin to Objects in Ruby, they are not. In fact, they are much closer to the Ruby Hashes we just reviewed. JavaScript objects, just like Ruby Hashes consist of key and value pairs where the keys in this case are called names and the value is called a value; together these form name-value pairs. JavaScript Objects are much less rigid than Ruby Hashes and can contain things called methods. These methods are functions that are stored in a name just like other values are. These methods can be called on the objects in which they are stored. Please see the code below for an example of a JavaScript Object.

            daniel_info = {
            home_town: "Palo Alto",
            full_name: "Daniel Kelley",
            age: 25,
            fav_food: "ice cream",
            yearsTilIm100: function(age){
              var til100 = 100 - this.age;
              alert("I will be 100 years old in " + til100 + " years.");
              }
            };

            console.log(daniel_info["full_name"]);      // "Daniel Kelley"
            console.log(daniel_info.age);               // 25
            daniel_info.yearsTilIm100();                // alerts "I will be 100 years old in 75 years."

At this point you might be asking yourself if JavaScript has classes because it is object oriented and everything. The short answer is 'no' JavaScript is what is known as a Prototype-based language which means that it achieves inheritance with something called, not surprisingly, prototypes.
